This guide walks you through cloning the repo, installing dependencies, and running your first research query.

## Prerequisites

Before you begin, make sure you have:

- **Python 3.13+** — the project requires `python>=3.13` (see `pyproject.toml`)
- **[uv](https://docs.astral.sh/uv/)** — the project uses `uv` for dependency management with `uv.lock`
- **Bright Data account** — you need a Bright Data API key for SERP and Reddit scraping
- **OpenAI API key** — used for GPT-4o LLM calls

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/balaji1233/DEEP_RESEARCH_AGENT.git
    cd DEEP_RESEARCH_AGENT
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    ```bash
    python -m venv .venv
    source .venv/bin/activate
    ```

    On Windows, activate with:

    ```bash
    .venv\Scripts\activate
    ```
  </Step>

  <Step title="Install dependencies with uv">
    The project declares its dependencies in `pyproject.toml` and locks them in `uv.lock`:

    ```toml pyproject.toml
    [project]
    name = "ai-search-agent"
    version = "0.1.0"
    requires-python = ">=3.13"
    dependencies = [
        "langchain>=0.3.27",
        "langchain-openai>=0.3.29",
        "langgraph>=0.6.4",
        "python-dotenv>=1.1.1",
    ]
    ```

    Install all dependencies:

    ```bash
    uv sync
    ```

    <Tip>
      `uv sync` installs from `uv.lock` for reproducible builds. Run `uv lock --upgrade` to update dependencies.
    </Tip>
  </Step>

  <Step title="Create your .env file">
    Create a `.env` file in the project root with your API keys:

    ```bash .env
    BRIGHTDATA_API_KEY=your_brightdata_api_key_here
    OPENAI_API_KEY=your_openai_api_key_here
    ```

    The agent loads these at startup via `python-dotenv`. See [Configuration](/configuration) for details on each variable.

    <Warning>
      Never commit your `.env` file to version control. Add it to `.gitignore` to keep your API keys safe.
    </Warning>
  </Step>

  <Step title="Run the agent">
    ```bash
    python main.py
    ```

    You should see:

    ```
    Multi-Source Research Agent
    Type 'exit' to quit

    Ask me anything:
    ```
  </Step>
</Steps>

## Example query and output flow

Once the agent is running, type a research question at the prompt:

```
Ask me anything: What are developers saying about LangGraph in 2025?
```

The agent initializes a typed state object and passes it to the LangGraph workflow:

```python
state = {
    "messages": [{"role": "user", "content": user_input}],
    "user_question": user_input,
    "google_results": None,
    "bing_results": None,
    "reddit_results": None,
    "selected_reddit_urls": None,
    "reddit_post_data": None,
    "google_analysis": None,
    "bing_analysis": None,
    "reddit_analysis": None,
    "final_answer": None,
}
final_state = graph.invoke(state)
```

You will see the following output as the agent works through the pipeline:

